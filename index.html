Who I am

Hello! My name is Nicolas and I am a junior Technical Game Designer studying at RUBIKA SupinfoGame in France.

I do Game Design

I like to design new games with unique mechanics and systems, finding inspiration from online multiplayer games and sandbox games to give the player unique ways to have fun. I am also used to working with Agile methods of production as I had the opportunity to use them in short and long term projects.

And programming

I enjoy programming game features and prototyping concepts to get a quick overview of the potential of the systems and mechanics but also identify the challenges in the production of games. I am entertained by working with Unity3D and C# and I am currently learning Unreal Engine's workflow with blueprints. Finally I usually create tools for my projects to facilitate the development of my games.

What am I good at?

Creative Ideas

I can design game systems and mechanics. Create documentation and share my work with everyone in the team. I am used to working with many people specialized in different aspects of game development.

Prototyping

For the past three years, I have been working with Unity3D and I really enjoy programming. I can create game prototypes but also work on project architecture and technical workflow.

Simple and useful tools

I have never done a project without creating tools. Extending the game engine editor I am working on always allowing the team I am working with to save time and keep their productivity to its maximum potential.

Communication between teams

I can easily communicate with different teams. I can especially produce technical documentation for game programmers to understand game features faster. I can also provide a technical point of view during the conception and anticipate technical difficulties.

Seance

About

        Role: Technical game designer, Lead programmer
        Duration: 9 months
        Team: 9 students
        Genre: Roguelike
        Platform: PC
        Engine: UnityEngine

Game Overview

Seance is an online multiplayer game in which three players will embody participants in a role-playing game and will have to outsmart the challenges of the game master.

The true nature of the game is revealed when a powerful demon, haunting the game, starts possessing the game master. Players will have no other choice but to cheat to make their way through the traps of the demon.

Watch out not getting caught, or you may regret your actions.

My Work

Seance is my graduation project.

In this project, I am in charge of the technical pipeline, the creation of the project architecture with its main systems and the development of tools for the game designers and game artists.

I started this project by writing the technical pipeline and share it with the team in the project's documentation on Notion. I am developing a complete project architecture to handle a multiplayer turn based card game with Fish-Networking and an event queue system. I also developed different tools like a card editor or a nodal tool to edit timed visual events outside of the code to ease the integration of visual feedback.

This project allows me to experiment a complete game development production for both technical game design and lead programming.

Fall Garden

About

        Role: Lead programmer, Game designer
        Duration: 5 months
        Team: 8 students
        Genre: Snake-like narrative game
        Platform: Mobile (Android)
        Engine: UnityEngine

Game Overview

Fall Garden is an initiatory fable on self-acceptance in an autumn colored forest.

The game is split into two phases. The hub allows the player to explore the town and learn more about the villagers.

The run phase is played with a snake-like controller where the player will have to defeat enemies and fight against the villagers mental illness.

My Work

My job on this project was to establish the programming pipeline and develop the main features of the game.

I started the project by writing a programming pipeline and sharing it with the rest of the team since all the game designers with me were also going to dive into the programming of game features.

I also developed the grid system and the character controller for both HUB and RUN phases.

This project allowed me to learn about mobile development challenges and the need for a good optimisation both for scripts and graphics.

Overstrike Arena

About

        Role: Project owner, Technical game designer
        Duration: 4 months
        Team: 7 students
        Genre: Fast FPS
        Platform: PC
        Engine: UnityEngine

Game Overview

Overstrike Arena is a 2vs2 multiplayer online competitive game. The player embodies a character with a first person view.

The games of Overstrike Arena are fast and nervous. The objective is simple: get the Overdrive and bring it to the enemy base to score a point.

The arenas in which players will compete evolve during the game, players with the best ability to adapt will take the lead.

Finally, close combat is at the heart of the experience and the best punch user, one of the core mechanics of the controller, will take the advantage.

My Work

I handled two roles on this project.

First of all, I am the project owner of Overstrike Arena so my job is to make sure that the gameplay intentions that the game should have are always respected. Moreover, during the 5 months of the project development, one of my responsibilities was to follow the work of the members of the team and check that the features work with each other.

When I am not working on the intentions of the game, my role is to be the technical game designer of the team. This means that I create all the technical documentation the programmer needs to develop the different features and I also help him by creating tools and developing minor aspects of the game to test new mechanics and win some time.
